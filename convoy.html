Build

Convoy — a full-stack field operations PWA

A mobile-first Progressive Web App for a veteran-owned medical courier company managing government-contract logistics — real-time run tracking, chain-of-custody proof, and an AI assistant built into the workflow. Designed, built, and shipped solo in four weeks.

Role

Solo — architecture, database, UI, AI integration, deployment

Timeline

4 weeks, in production use

Stack

React, Vite, Supabase, Claude API, Vercel

Why this exists

The client is a 100% service-disabled veteran who owns a medical courier company holding government contracts through SAM.gov (VA, DoD, HHS). Field operations — run assignment, chain-of-custody proof, compliance tracking — were running on paper and text messages, which doesn't hold up under a government contract's audit requirements. Convoy replaces that with a role-based mobile app dispatchers and drivers actually use in the field.

What it does

Field operations

Real-time run tracking (Supabase Realtime), camera capture with GPS tagging, touch signature capture, and a chain-of-custody event log written on every status change.

Works without signal

Offline queue backed by IndexedDB, syncs automatically on reconnect — built for drivers in low-connectivity areas, not just a happy-path demo.

AI assistant + anomaly detection

A Supabase Edge Function proxies the Claude API for an in-app assistant and flags runs running >40% over average duration — AI integrated into a real operational workflow, not a chat widget bolted on.

Compliance automation

SAM.gov contract expiry alerts, SDVOSB credential tracking, an AI-scored contract-opportunity matcher, and branded chain-of-custody PDF generation for audits.
